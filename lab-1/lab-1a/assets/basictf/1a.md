Why Secrets Manager Is Preferred

Using AWS Secrets Manager provides:

Centralized secret lifecycle management

Fine-grained IAM authorization

Encryption at rest

Audit logging via CloudTrail

Runtime retrieval (no hardcoding)

Optional automatic rotation

Risks of Hardcoding Credentials

Storing credentials in:

User data

Environment variables

Source code repositories

introduces risks including:

Credential leakage

Lack of auditability

No rotation capability

Increased blast radius

Why Broad Access Is Restricted

Role Justification

The IAM role:

Provides a secure identity for the EC2 instance

Eliminates hardcoded credentials

Enables secure AWS API access

Supports credential rotation strategies

Scoped Secret Access

Access is granted via an identity policy scoped to the specific secret.

Security Benefits:

Resource-level permission control

Isolation between secrets

Reduced attack surface
